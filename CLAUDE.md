## Project Overview

METICS is a course enrollment and training management system for faculty and administrative staff at Universidad de Costa Rica (UCR). It tracks participant enrollment in workshops/courses, hours completed, grades, and advisor assignments.

## Commands

```bash
# Build
dotnet build

# Run (development)
dotnet run --project webMetics --launch-profile https   # https://localhost:7201
dotnet run --project webMetics --launch-profile http    # http://localhost:5065

# Restore dependencies
dotnet restore
```

There are no automated tests in this project.

## Database Setup

1. Install SQL Server and create a database named `SIMETICS`
2. Run the scripts in order:
   - `db_scripts/db_METICS.sql` — main schema (run first on a fresh DB)
   - `db_scripts/db_Carreras.sql`
   - `db_scripts/db_UpdateParticipante_AreasExtras.sql`
   - `db_scripts/db_UpdateGrupo_fix.sql`
   - `db_scripts/bitacora_db.sql`
3. Update the connection string in `webMetics/appsettings.json`

Email sending is disabled by default (`EmailSettings:IsEnabled: false`). Set to `true` and configure SMTP credentials to enable it.

## Architecture

**ASP.NET Core 8.0 MVC** with Razor views. No ORM — all data access uses raw ADO.NET (`SqlCommand`/`DataTable`).

### Data Access Pattern (Handler Pattern)

Every domain area has a dedicated Handler class in [webMetics/Handlers/](webMetics/Handlers/) that inherits from `BaseDeDatosHandler`. Handlers own all SQL queries and map `DataTable`/`SqlDataReader` results to model objects manually. Controllers instantiate handlers directly (no dependency injection for handlers).

```
Controller → Handler → SQL Server (SIMETICS database)
                     ← DataTable → manual mapping → Model
```

### Authentication

Custom auth — no ASP.NET Core Identity. Passwords stored as `BINARY(64)` hash + `UNIQUEIDENTIFIER` salt in the `usuario` table.

**Roles** (stored in session/ViewBag as `ViewBag.Role`):
- `0` — Participant
- `1` — Admin  
- `2` — Advisor (Asesor)

Role checks are done manually in controllers and views via `ViewBag.Role`. `IDataProtectionProvider` is used for protecting sensitive values in URLs.

### Key Domain Concepts

- **Grupo** — a course/workshop instance with enrollment windows, modality, capacity, and one or more associated themes (`TemaHandler`)
- **Participante** — a UCR employee who can enroll; tracks cumulative enrolled hours and completed hours
- **Inscripcion** — the enrollment record linking a participant to a grupo
- **Asesor** — an instructor/facilitator who leads grupos
- **Calificacion** — grade/completion record per participant per grupo

### Academic Area Hierarchy

`wwwroot/data/dataAreas.json` holds the full hierarchical structure: **Areas → Departments → Sections → Careers** (keyed by campus). This file is generated by `scripts/carreras/generar_carreras.py` from a CSV of academic offerings — do not edit it manually. See `scripts/carreras/README.md` for regeneration instructions.

### Export Capabilities

Controllers use **EPPlus/NPOI** for Excel exports and **iText** for PDF generation. These are invoked directly from controller action methods that return `FileResult`.

### Frontend

Bootstrap 5.3.2 + jQuery. CSS/JS is bundled/minified via `bundleconfig.json` (BundlerMinifier). Shared layout is in [webMetics/Views/Shared/_Layout.cshtml](webMetics/Views/Shared/_Layout.cshtml). ViewBag is used extensively to pass data from controllers to views.
